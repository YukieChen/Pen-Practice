> **設計原則**: Jony Ive 極簡主義 — 移除一切不必要的裝飾  
> **對應流程**: Phase 2 — Architecture & Planning

---

## 1. 設計理念

### 1.1 核心原則

- **一個目標**: 使用者從開啟網頁到取得 PDF，步驟越少越好
- **零學習成本**: 介面自解釋，不需要說明書
- **內容即介面**: 字帖的預覽就是最好的 UI

### 1.2 設計約束

- 單頁應用 (SPA)，不需路由
- 響應式設計：桌面優先，支援平板（手機非主要目標，因列印需求以桌機為主）
- 無需帳號系統、無需儲存狀態

```
│  │  ...                                                │  │

│                                                            │
│  ┌─ 第 4 段 ─────────────────────────────────────────┐  │
│  │  ...                                                │  │
│  └─────────────────────────────────────────────────────┘ │
│                                                            │
│  統計: 共 4 段, 24 字 │ 有說明: 1 頁 │ 無說明: 2 頁       │
│                                                            │
└──────────────────────────────────────────────────────────┘

設計:
- 每段為可收合的卡片 (accordion)
- 每段包含: 說明 input + 練習字 input
- 說明 input: 「需要說明列」時顯示; 「不需要說明列」時隱藏
- 練習字 input: 單行 text input, 自動過濾非中文字元，**最多接收 64 個字**；超出部分自動截斷並在段落下方顯示提醒
- 每段顯示字數統計 + 模式提示 (>4字時顯示隨機挑字/全部印出提示)
- 可動態增減段落 (「+ 新增段落」/ 「✕ 移除」按鈕)，**最多 200 段**；嘗試新增第 201 段時給予提示並阻止新增
- 最少 1 段（至少保留一段不能刪除，避免全空狀態）
- 底部顯示全局統計: 總段數、總字數、預估頁數 (依模式計算)

> 安全顯示原則：所有顯示使用者輸入或設定檔載入的文字（說明、練習字、統計、提醒），一律以純文字方式渲染，不插入 HTML。
```

## 5. 狀態管理

### 5.1 應用狀態

```
AppState {
  mode:             'no-description' | 'with-description'
  contentSource:    'manual' | 'upload' | 'preset'

  // 段落內容 (核心資料模型)
  sections:         SectionInput[]     // 動態長度 (最少 1 段)

  fontSource:       'default' | 'custom'
  customFontFile:   File | null
  customFontName:   string | null
  parsedFont:       opentype.Font | null

  // 衍生狀態 (computed)
  totalChars:       number     // 所有段落的總字數
  effectiveChars:   number     // 依模式計算的實際印出字數
  pageCount:        number     // 依模式計算的頁數
  isReady:          boolean    // totalChars > 0
  isGenerating:     boolean
  progress:         number     // 0-100
}

SectionInput {
  description:      string     // 說明文字 (可為空)
  characters:       string     // 過濾後的中文字串 (可 >4 字)
}
```

### 5.2 狀態轉換

```
初始 ──→ [選擇模式] ──→ 模式已選
              │
              ▼
        [選擇內容來源] ──→ 手動: 直接到段落編輯
              │             上傳: 解析 JSON → 填入段落
              │             預設: 選擇 → 填入段落
              ▼
        [編輯段落] ──→ 準備就緒 (isReady = true)
              │
              ▼
        [上傳字體] ──→ 字體解析中 → 字體就緒
              │         (可選)        │
              │                       │
              ▼                       ▼
        [點擊產生/下載] ──→ 產生中 ──→ 完成 (下載 PDF)
              │                            或: 下載 JSON
              ▼ (失敗)
           錯誤提示
```

## 9. 動畫與過渡

| 觸發 | 效果 | 時長 |
|------|------|------|
| 模式切換 | 說明欄位展開/收合 | 300ms ease-out |
| 卡片選擇 | 邊框色 + 背景色過渡 | 200ms ease |
| 內容來源切換 | Tab 內容淡入淡出 | 200ms ease |
| 段落新增 | 卡片滑入 | 300ms ease-out |
| 段落移除 | 卡片收合消失 | 200ms ease-in |
| 設定檔/預設載入 | 段落內容閃爍填入 | 150ms |
| 字體上傳 | 進度條填充 | 依實際時間 |
| 產生按鈕按下 | 按鈕寬度擴展為進度條 | 300ms ease |
| PDF 完成 | 按鈕閃綠色 + 勾號 | 200ms, 2s 後復原 |
