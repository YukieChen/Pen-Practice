## 1. 產品概述

### 1.1 產品定義

**鋼筆字字帖產生器** — 一個純前端 Web 應用程式，讓使用者輸入中文字與上傳字體檔，自動產生可列印的 A4 鋼筆字練習字帖 PDF。

### 1.2 核心價值

- 使用者可以用**任意字體**產生練習字帖，不限於預設字體
- 支援「有說明列」與「無說明列」兩種格式
- 支援 **JSON 設定檔**上傳/下載，可重複使用與分享字帖配置
- 內建**系統預設設定檔**，提供常見書法結構分類範本
- 瀏覽器內完成所有運算，無需後端伺服器

### 1.3 目標使用者

練習鋼筆字（硬筆書法）的中文使用者，需要客製化字帖進行描寫與臨摹練習。

## 3. 格線規格 (Grid Specification)

本節定義字帖格線的精確數學規格。所有尺寸為最終 PDF 輸出尺寸。

### 3.1 紙張與邊距

| 屬性 | 值 |
|------|------|
| 紙張尺寸 | A4 (210mm x 297mm) |
| fine-cell 尺寸 | 3.7mm x 3.7mm (正方形) |
| 格線區域寬度 | 3.7mm x 48 = 177.6mm |
| 格線區域高度 | 3.7mm x 72 = 266.4mm |
| 左右邊距 | (210 - 177.6) / 2 = 16.2mm |
| 上下邊距 | (297 - 266.4) / 2 = 15.3mm |

### 3.2 格線階層

格線分為三個視覺層次：

```
┌───────────────────────────────────────────────┐
│ 層次 1: Fine Grid (精細格線)                    │
│   尺寸: 72 rows x 48 cols                      │
│   線寬: 0.1pt                                   │
│   顏色: #000000, opacity 10%                    │
│   用途: 輔助筆劃比例參考                        │
│                                                 │
│ 層次 2: Character Cell Grid (字格格線)          │
│   尺寸: 每 3x3 fine-cells 邊界                  │
│   等效: 24 rows x 16 cols                       │
│   線寬: 0.3pt                                   │
│   顏色: #000000, opacity 40%                    │
│   用途: 定義每個字的書寫區域                    │
│                                                 │
│ 層次 3: Section Divider (段落分隔線)            │
│   位置: 每 6 字行 (18 fine-rows) 的水平邊界     │
│   線寬: 0.5pt                                   │
│   顏色: #000000, opacity 60%                    │
│   用途: 區隔不同練習段落                        │
└───────────────────────────────────────────────┘
```

### 3.3 字格矩陣

```
總格線:    72 fine-rows  x  48 fine-cols
字格矩陣:  24 char-rows  x  16 char-cols
字格尺寸:  11.1mm x 11.1mm (3 x 3 fine-cells)
```

### 3.4 水平分組 (Character Groups)

16 字欄分為 4 個字組 (Character Group)，每組 4 字欄：

```
←─────── 48 fine-cols (177.6mm) ───────→

┌──────────┬──────────┬──────────┬──────────┐
│  字組 A   │  字組 B   │  字組 C   │  字組 D   │
│ 4 字欄    │ 4 字欄    │ 4 字欄    │ 4 字欄    │
│ 12 fine-c │ 12 fine-c │ 12 fine-c │ 12 fine-c │
└──────────┴──────────┴──────────┴──────────┘

每個字組內部結構:
┌────────┬────────┬────────┬────────┐
│ 欄 1   │ 欄 2   │ 欄 3   │ 欄 4   │
│ 描寫欄 │ 描寫欄 │ 臨摹欄 │ 臨摹欄 │
│ 淺色字 │ 淺色字 │ 空格線 │ 空格線 │
└────────┴────────┴────────┴────────┘
```

- **欄 1-2 (描寫欄)**: 顯示淺色引導字，使用者在上方描寫
- **欄 3-4 (臨摹欄)**: 僅顯示空白格線，使用者看著範字臨摹

### 3.5 垂直分段 (Sections)

24 字行分為 4 個練習段 (Section)，每段 6 字行：

```
←─── 24 char-rows (72 fine-rows) ───→

┌─────────────┐ ─┐
│  Section 1   │  │ 6 char-rows = 18 fine-rows
│  字: A1~A4   │  │ 練習字組 A 的 4 個字
└─────────────┘ ─┘
┌─────────────┐ ─┐
│  Section 2   │  │ 6 char-rows
│  字: B1~B4   │  │ 練習字組 B 的 4 個字
└─────────────┘ ─┘
┌─────────────┐ ─┐
│  Section 3   │  │ 6 char-rows
│  字: C1~C4   │  │
└─────────────┘ ─┘
┌─────────────┐ ─┐
│  Section 4   │  │ 6 char-rows
│  字: D1~D4   │  │
└─────────────┘ ─┘
```

每頁共練習 **4 段 x 4 字/段 = 16 個不同的字**。

## 4. 兩種模式規格

### 4.1 核心概念：段落字池 (Section Character Pool)

每個段落 (Section) 包含一組**字池 (Character Pool)**，字池中的字數可 >= 1，不限於 4 個：

```
SectionConfig {
  description: string      // 書法結構說明 (可為空)
  characters:  string[]    // 字池 (1~N 個字，不限 4 個)
}
```

兩種模式對字池的處理方式不同：

```
┌───────────────┬─────────────────────────────────────────┐
│ 模式           │ 字池處理規則                             │
├───────────────┼─────────────────────────────────────────┤
│ 有說明列       │ 每段最多取 4 字 (字池 >4 則隨機挑選)    │
│               │ 說明列顯示 description 文字              │
│               │ 每頁最多 4 段，超過分頁                  │
├───────────────┼─────────────────────────────────────────┤
│ 無說明列       │ 忽略 description                        │
│               │ 所有段落的字池合併為扁平清單              │
│               │ 所有字全部印出，每頁 16 字，超過分頁     │
└───────────────┴─────────────────────────────────────────┘
```

### 4.2 模式一：無說明列

**字的來源**: 合併所有段落字池為扁平清單，全部印出。

```
範例:
  Section 1: chars = ["寅","宙","守","宿","寶","家","窩","宰"]
  Section 2: chars = ["至","盆","盤","鹽"]
  
  合併: ["寅","宙","守","宿","寶","家","窩","宰","至","盆","盤","鹽"]
  共 12 字 → 1 頁 (4段 x 4字/段 = 16 容量, 第4段部分空白)
```

每段 6 字行全部用於練習：

```
Section 結構 (無說明列):

行 1: [範字][引導][格線][格線] │ [範字][引導][格線][格線] │ ... (4 組)
行 2: [引導][引導][格線][格線] │ [引導][引導][格線][格線] │ ...
行 3: [引導][引導][格線][格線] │ [引導][引導][格線][格線] │ ...
行 4: [引導][引導][格線][格線] │ [引導][引導][格線][格線] │ ...
行 5: [引導][引導][格線][格線] │ [引導][引導][格線][格線] │ ...
行 6: [引導][引導][格線][格線] │ [引導][引導][格線][格線] │ ...

練習量/字: 描寫 (6行x2欄=12次) + 臨摹 (6行x2欄=12次) = 24 次
```

### 4.3 模式二：有說明列

**字的來源**: 每段從字池中取字，字池 > 4 字時隨機挑選 4 字。

```
範例:
  Section 1: chars = ["寅","宙","守","宿","寶","家","窩","宰"]
  → 字池 8 字 > 4，隨機挑 4 字，如 ["宿","家","窩","宰"]
  
  Section 2: chars = ["至","盆","盤","鹽"]
  → 字池 4 字 = 4，全部使用
```

每段第 1 行融合為說明列，剩餘 5 行用於練習：

```
Section 結構 (有說明列):

行 1: ┌──────────────────────────────────────────────┐
      │ 說明文字 (跨全寬 48 fine-cols, 高 3 fine-rows) │
      └──────────────────────────────────────────────┘
行 2: [範字][引導][格線][格線] │ [範字][引導][格線][格線] │ ... (4 組)
行 3: [引導][引導][格線][格線] │ [引導][引導][格線][格線] │ ...
行 4: [引導][引導][格線][格線] │ [引導][引導][格線][格線] │ ...
行 5: [引導][引導][格線][格線] │ [引導][引導][格線][格線] │ ...
行 6: [引導][引導][格線][格線] │ [引導][引導][格線][格線] │ ...

練習量/字: 描寫 (5行x2欄=10次) + 臨摹 (5行x2欄=10次) = 20 次
```

### 4.4 說明列規格

| 屬性 | 值 |
|------|------|
| 高度 | 1 字行 = 3 fine-rows = 11.1mm |
| 寬度 | 全頁寬 = 48 fine-cols = 177.6mm |
| 背景 | 無 (透明) |
| 文字大小 | 自動適配行高，約 9-10pt |
| 文字顏色 | #000000, opacity 80% |
| 文字對齊 | 靠左對齊，左方留 2mm padding |
| 字體 | 與練習字相同的 Glyph Path 渲染字體來源 (使用者上傳字體；未上傳時使用內建預設字體) |
| 內容格式 | 純文字，可含編號前綴 (如 "01、") |

**安全渲染規範**：

- 說明列與其他所有使用者輸入/設定檔載入的文字內容，在前端顯示時一律以「純文字節點」渲染，不得透過 `innerHTML` 等方式插入 HTML，以避免 XSS 或惡意 JSON 設定檔注入腳本。
- JSON 設定檔中的字串內容在匯入後，顯示於 UI 的所有位置也套用相同規則。

### 4.5 頁面填滿規則

無論字數是否足以填滿整頁，格線永遠畫滿一頁 (72x48 fine grid)。空白字組僅顯示格線，供使用者自行運用。

## 5. 字元渲染規格

### 5.1 渲染方式

採用 **Glyph Path 直接繪製** 方式：

1. 使用 opentype.js 解析字體檔，提取目標字元的向量路徑 (glyph path)
2. 計算路徑的 Ink Bounding Box (實際筆劃邊界)
3. 數學變換 (Scale + Translate) 將筆劃精準縮放並置中於目標字格
4. 將變換後的向量路徑直接繪製到 PDF

此方式完全消除不同字體的 metrics 差異，確保任何字體的筆劃都精準填滿字格。

### 5.1.1 缺字處理機制

當使用者目前選用的字體缺少某些字元的 glyph 時，系統**不再以其他字體自動替代**，而是保留空白練習格，並清楚提示缺字狀態：

```
渲染字元 "曦":
  1. 檢查目前選用字體是否含 "曦" 的 glyph
  2. 若有 → 使用該字體的 glyph path 正常渲染
  3. 若無 → 不繪製任何筆劃，該格僅顯示 3x3 格線 (留白練習格)
```

- **預設字體**: 應用程式內建一套可合法分發的 CJK 書寫用字體，做為「未上傳字體時」的主字體來源
- **缺字行為**: 任何選用字體（內建或自訂）遇到缺字時，一律採「留白格」策略，不自動 fallback 至其他字體
- **使用者提示**: 缺字時顯示警告訊息，列出哪些字在字帖中將顯示為空白格，讓使用者清楚知道是「字體缺字」而非系統故障

### 5.2 字元類型與渲染參數

#### 範字 (Model Character)

- **位置**: 每段第 1 練習行、每組第 1 欄
- **填充色**: #000000
- **opacity**: 100%
- **縮放**: 筆劃填滿字格的 90% (padding = 5% each side)
- **用途**: 書寫範本，展示目標字的完整筆劃

#### 引導字 (Guide Character)

- **位置**: 描寫欄 (欄 1-2) 的所有格子 (範字位置除外)
- **填充色**: #AAAAAA
- **opacity**: 25%
- **縮放**: 與範字相同 (筆劃填滿字格的 90%)
- **用途**: 淺色引導，使用者在上方描寫

#### 空白格 (Blank Cell)

- **位置**: 臨摹欄 (欄 3-4) 的所有格子
- **內容**: 僅顯示格線，無字元
- **用途**: 使用者看著範字自行臨摹

### 5.3 Glyph Path 縮放演算法

```
輸入:
  cellSize   = 11.1mm                    (字格尺寸)
  fillRatio  = 0.90                      (填充比例)
  targetSize = cellSize * fillRatio      (= 9.99mm)
  padding    = cellSize * (1 - fillRatio) / 2  (= 0.555mm)

步驟:
  1. glyph = font.charToGlyph(character)
  2. bbox  = glyph.getBoundingBox()       → { x1, y1, x2, y2 } (font units)
  3. glyphW = bbox.x2 - bbox.x1
     glyphH = bbox.y2 - bbox.y1
  4. scale = min(targetSize / glyphW, targetSize / glyphH)
  5. scaledW = glyphW * scale
     scaledH = glyphH * scale
  6. offsetX = cellOriginX + (cellSize - scaledW) / 2
     offsetY = cellOriginY + (cellSize - scaledH) / 2
  7. 對路徑每個控制點:
     newX = (pathX - bbox.x1) * scale + offsetX
     newY = (pathY - bbox.y1) * scale + offsetY
```

## 6. 使用者輸入規格

### 6.1 模式選擇

- **輸入方式**: 二選一按鈕 (toggle / radio)
- **選項**: 「需要說明列」/「不需要說明列」
- **預設值**: 「不需要說明列」

### 6.2 內容輸入 (三種來源)

使用者可透過以下三種方式提供練習內容，三者互通（任一方式填入後，其他方式可見/可編輯相同資料）：

#### 6.2.1 手動輸入 (Per-Section)

UI 提供 4 個段落輸入區塊（對應每頁 4 個 Section），每個區塊包含：

- **說明文字**: 單行 input (「需要說明列」模式才顯示)
  - 字數限制: 最長 50 字
  - 可留空
- **練習字**: 文字輸入框
  - 接受字元: CJK Unified Ideographs (U+4E00–U+9FFF, Extension A-F)
  - 自動過濾非中文字元
  - **可輸入超過 4 個字** (字池概念)
  - 最少 1 字/段 (空段自動跳過)

使用者可動態增減段落數量（最少 1 段）。

#### 6.2.2 上傳 JSON 設定檔

- **接受格式**: `.json`
- **上傳後**: 自動解析並填入各段落的說明 + 練習字
- **JSON 格式**:

```json
{
  "version": 1,
  "sections": [
    {
      "description": "01、天覆者凡畫皆冒於其下：上面是寶蓋的字，其餘筆畫應冒於其下。",
      "characters": ["寅", "宙", "守", "宿", "寶", "家", "窩", "宰"]
    },
    {
      "description": "02、地載者有畫皆托於其上：下面有底托狀的字，其餘筆畫應托於其上。",
      "characters": ["至", "盆", "盤", "鹽"]
    }
  ]
}
```

- **version 欄位**: 整數版本號，省略時視為 `1`，未來格式演進時用於向下相容處理
- **characters 欄位**: 每段可超過 4 個字 (完整保存字池)
- **說明長度**: 載入時每段 `description` 最多保留前 50 字，多餘部分自動截斷（行為與 UI 輸入一致）
- **上傳後可編輯**: 填入後使用者仍可手動修改
- **驗證**: 檢查 JSON 結構合法性，`sections` 必須為陣列，`characters` 必須為中文字元陣列

#### 6.2.3 設定檔資料量上限

為避免惡意或意外的大型設定檔拖垮瀏覽器，系統對匯入資料量施加上限：

- **最大段落數**: 預設上限 200 段，超過部分自動截斷（僅載入前 200 段）
- **每段最大字池**: 預設上限 64 字，超過部分自動截斷（保留前 64 字）
- 超出上限時：
  - 不視為錯誤，不中斷匯入流程
  - 於 UI 顯示提醒：「設定檔內容過多，已截斷為前 N 段／每段前 M 字」

#### 6.2.4 選擇系統預設設定檔

- **UI**: 按鈕群組 (button grid)，每批次一個按鈕
- **選擇後**: 自動填入對應批次的段落說明 + 練習字
- **選擇後可編輯**: 填入後使用者仍可手動修改
- **預設內容**: 見 §6.5 系統預設設定檔

#### 6.2.5 輸入內容與 JSON 檔案的整體上限

為避免惡意或意外的大型內容拖垮瀏覽器，對所有來源（手動輸入、上傳設定檔、系統預設）採用一致上限：

- **JSON 檔案大小上限**: 上傳設定檔大小上限為 5MB；超過時直接拒絕解析，顯示錯誤訊息提示使用者重新檢查檔案內容
- **最大段落數**: 手動輸入與設定檔載入後的段落總數上限為 200 段；UI 將阻止新增超過 200 段
- **每段最大字池**: 單一段落的字池上限為 64 字；超過時僅保留前 64 字，其餘自動截斷並顯示提醒

### 6.3 下載設定檔

- **功能**: 將當前 UI 中的所有段落（說明 + 完整字池）匯出為 JSON 檔案
- **下載格式**: 與上傳格式相同 (見 §6.2.2)
- **字池完整保存**: 即使 > 4 字，下載時全部保存（不做截斷）
- **檔案名稱**: `字帖設定_YYYYMMDD_HHmmss.json`
- **位置**: 與「產生 PDF」按鈕並列

### 6.4 字體上傳

- **輸入方式**: 檔案上傳元件 (file input)
- **接受格式**: .ttf, .otf
- **檔案大小限制**: 最大 20MB
- **預設字體**: 應用程式內建一套可合法分發的 CJK 書寫用字體，做為「未上傳字體時」的主字體
  - 未上傳字體時：使用此預設字體作為主字體
  - 上傳字體後：以使用者上傳字體作為主字體；若該字體缺字，對應格子留白，不以其他字體自動替代
- **驗證**: 上傳後驗證字體檔是否可被 opentype.js 正常解析
- **字元覆蓋檢查**: 驗證字體是否包含所有練習字的 glyph；缺字時提示警告，列出缺字清單，說明對應格子將為空白格線

## 7. PDF 輸出規格

### 7.1 檔案屬性

| 屬性 | 值 |
|------|------|
| 格式 | PDF 1.7+ |
| 紙張 | A4 (210mm x 297mm) |
| 方向 | 直式 (Portrait) |
| 色彩空間 | Grayscale |
| 解析度 | 向量圖形 (無限縮放) |

### 7.2 頁面結構

每頁包含以下繪製層 (由下而上):

```
Layer 1 (底層): Fine Grid — 72x48 淺色格線
Layer 2 (中層): Character Cell Grid — 24x16 字格邊框
Layer 3 (中層): Section Dividers — 段落分隔線
Layer 4 (上層): Description Text — 說明文字 (僅有說明列模式)
Layer 5 (上層): Guide Characters — 引導字向量路徑 (淺色)
Layer 6 (最上): Model Characters — 範字向量路徑 (深色)
```

### 7.3 多頁規則

**無說明列模式**:
- 所有段落字池合併為扁平清單
- 每 16 字一頁，超過自動分頁
- 最後一頁不足 16 字時，空白字組僅顯示格線（頁面格線永遠畫滿）

**有說明列模式**:
- 每頁最多 4 段（每段 1 說明 + 4 字）
- 字池 > 4 字的段落：隨機挑選 4 字
- 超過 4 段時自動分頁
- 最後一頁不足 4 段時，空白段落僅顯示格線（頁面格線永遠畫滿）

**共通規則**: 所有頁面格線結構相同 (72x48 fine grid)，永遠畫滿一頁。

## 8. 非功能需求

### 8.1 瀏覽器相容性

- **目標**: 現代瀏覽器 (Chrome 90+, Firefox 90+, Safari 15+, Edge 90+)
- **必要 API**: File API, Blob API, ArrayBuffer
- **不依賴**: FontFace API (因採用 Glyph Path 方式，不需瀏覽器載入字體)

### 8.2 效能

- **PDF 產生時間**: 單頁 < 3 秒，10 頁 < 10 秒
- **記憶體**: < 200MB (含字體檔解析)
- **字體解析**: 首次解析 < 2 秒 (20MB 字體檔)

### 8.3 無障礙

- **鍵盤操作**: 所有輸入可用鍵盤完成
- **ARIA 標籤**: 所有互動元件提供適當 ARIA 屬性
- **色彩對比**: UI 介面符合 WCAG 2.1 AA 標準

## 9. 限制與假設

### 9.1 假設

- 使用者擁有合法授權使用其上傳的字體檔
- 使用者有可用的印表機 (A4 紙張)
- 目標瀏覽器支援 ES2020+ 語法

### 9.2 已知限制

- 不支援直式書寫 (僅橫式排列)
- 不支援彩色字帖 (僅灰階)
- 不支援自訂格線尺寸 (固定 72x48 / 3x3)
- 說明列文字不支援富文字格式 (僅純文字)
- 部分罕用字 (CJK Extension B+) 可能不被所有字體支援

### 9.3 未來擴充可能

- 支援自訂格線密度 (如 4x4, 5x5)
- 支援直式書寫排列
- 支援「十字輔助線」或「米字輔助線」於字格內
- 擴充更多系統預設設定檔 (如其他書法教材)
- 支援字帖預覽 (Canvas 即時預覽)
- 支援設定檔線上分享 (URL 參數)
